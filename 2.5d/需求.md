# Antigravity Interactive Avatar Experience（Google I/O 2026 Demo）
## 一、整体概念
基于 **Google Antigravity SDK** 的**像素风多 Agent 虚拟社交 Demo**。用户上传照片/自拍生成 16-bit 像素数字分身，进入一个**等轴测（斜45°）网格虚拟空间**；通过手机/电脑端的任务面板给分身下达自然语言指令，分身会**自主寻路、与其他 AI 驱动的数字分身发起对话互动**，所有角色行为、对话均由 Gemini 模型实时生成。

---

## 二、核心功能描述（对齐现场 Demo）
### 1. 数字分身生成（用户端）
- **生成方式**：上传一张自拍/头像 → AI 生成 16-bit 像素风格数字分身，支持多套外观变体选择（发型、服装配色等）。
- **人设配置**：用户可手动输入简短人设描述（如性格、爱好、说话风格），作为分身的对话行为基础。
- **无社交数据爬取**：人格完全由用户现场配置 + 模型实时生成，不读取任何第三方社交账号数据。
- **分身特征**：
  - 等轴测视角像素小人，带有基础的走路、站立、转向帧动画
  - 头像下方显示用户名（如 `Punkchou` / `Samuel`），底部带有彩色光环标识
  - 支持基础表情、挥手等简单互动动画

### 2. 虚拟空间（主场景）
- **视角与布局**：
  - 采用**等轴测斜45°视角**，而非纯俯视视角，空间呈现出轻微的3D立体感
  - 地面为全覆盖的正方形网格，线条为浅蓝/深灰线框，每个格子大小一致，角色严格吸附在格子上移动
- **空间元素**：
  - 背景：深蓝色渐变星空，点缀白色星点，营造科技感
  - ~~场景道具：蓝色圆形柱状物体（类似虚拟展台/桌子），分布在空间中作为互动点~~（已移除）
- **寻路与碰撞**：
  - 所有角色基于网格寻路，移动时沿格子路径行走，可绕开障碍物和其他角色
  - 角色移动为逐格平滑过渡，带有轻微的像素风移动动画
  - AI角色寻路时自动避开其他静止角色占用的格子，移动中的角色不视为占用

### 3. 角色与互动行为
#### 用户分身
- **控制方式**：通过手机/电脑端的任务面板下达指令，而非摇杆控制；同时支持鼠标左键拖拽移动视角、滚轮缩放
- **指令类型（与 Demo 一致）**：
  - 任务指令：`meet Samuel — Deep dive on Lobster Pass build details`、`Mingle at Maker Space`、`Wander`
  - 状态反馈：面板显示分身状态（如 `On the move` / `Arrived at Event Space`）
- **互动逻辑**：分身收到指令后，自主寻路到目标位置/目标角色旁，自动转向并发起对话，弹出对话气泡

#### AI 数字分身（其他角色）
- **行为模式**：空间内常驻多个 AI 驱动的数字分身，会随机走动、在场景中徘徊，也会主动与其他分身互动
  - AI每2-6秒随机选择行动（50%概率移动，20%静止，15%挥手，15%聊天）
  - AI移动速度：600-1000ms/格（较慢的休闲 pace）
- **对话互动**：两个分身靠近后，会弹出蓝色背景的对话气泡，气泡内显示双方的对话内容，对话由模型基于人设和场景实时生成
- **标识**：每个分身带有用户名光环，部分分身带有职业/角色标识（如 `ANTARIS-MK3` 等）

### 4. 用户控制端（手机/电脑）
- **界面布局**：
  - 任务面板：列出可执行的任务选项，支持单选，带有状态标识（绿色圆点表示正在执行）
  - 分身状态区：显示分身当前状态（如 `On the move` / `Arrived`）和任务详情
  - 生成界面：上传照片/头像后，生成多个像素分身变体供用户选择，同时显示人设输入框
- **交互流程**：
  1. 上传照片 → 生成分身 → 选择外观 + 配置人设
  2. 进入虚拟空间 → 选择任务指令 → 分身开始执行
  3. 实时查看分身状态和对话内容，可随时切换任务指令

## 三、UI 视觉细节（按 Demo 画面还原）
### 1. 虚拟空间 UI
- **网格地板**：深蓝色背景 + 浅蓝线框正方形网格，线条纤细，网格透视效果随等轴测视角变化
- **角色风格**：
  - 16-bit 像素风，Q版比例，线条清晰，颜色低饱和
  - 每个角色脚下带有蓝色/彩色光环，光环上显示白色用户名
  - 角色带有简单的行走帧动画（左右交替）和转向动画
- **对话气泡**：
  - 样式：半透明蓝色背景 + 白色文字，矩形框带轻微圆角，边缘有像素描边
  - 文字：白色像素字体，内容简短，显示说话者用户名 + 对话内容
  - 位置：气泡位于角色上方，带有轻微的指向箭头，显示时间约3-5秒

### 2. 用户控制端 UI
- **分身生成界面**：
  - 左侧：照片上传区 + 人设输入文本框，深色背景
  - 右侧：分身变体预览区，展示多个不同外观的像素分身，供用户选择
- **任务面板界面**：
  - 深色半透明背景，带有任务列表，每个任务带有单选按钮和状态标识
  - 顶部显示分身头像和当前任务标题，底部显示任务详情
- **整体风格**：深色科技风，低饱和配色，文字为无衬线/像素字体，与虚拟空间风格统一

### 3. 动效与过渡
- **角色移动**：沿网格路径平滑移动，每格移动约0.6-1.0秒（AI），玩家约0.25秒，带有轻微的上下起伏动画
- **气泡动效**：淡入 → 停留 → 淡出，无复杂特效，保持像素风简洁感
- **指令执行**：分身先转向目标方向 → 寻路移动 → 到达后弹出对话气泡，步骤清晰，反馈明确
- **视角控制**：鼠标左键拖拽平移，滚轮缩放，平滑过渡

## 四、H5 复刻关键要点（避坑）
1. **视角还原**：必须使用**等轴测斜45°视角**，而不是纯俯视，网格和角色要带有透视效果
2. **网格吸附与寻路**：角色严格吸附网格移动，寻路要能绕开场景道具和其他角色
3. **对话气泡样式**：蓝色半透明背景 + 白色文字，带有用户名标识，位置在角色上方
4. **控制方式**：核心是**指令驱动**，而非手动摇杆，重点实现任务面板和分身状态反馈
5. **多角色互动**：空间内需要多个AI分身，实现随机走动和自动对话，还原热闹的多Agent氛围
6. **角色占用机制**：静止角色占用所在格子，移动中角色不占用，寻路需实时更新占用状态

## 五、现实地图转网格方案（国内环境）

### 1. 推荐数据源：高德地图开放平台
- **官网**：[高德开放平台](https://lbs.amap.com/)
- **优势**：国内覆盖最全、API稳定、免费额度充足（个人日10万次）
- **注册**：申请账号 → 创建应用 → 获取 `Key`

### 2. 数据获取方式

#### 方式一：行政区边界查询（免费）
```javascript
const opts = {
  subdistrict: 0,
  extensions: 'all',  // 必须设为'all'才能拿到边界坐标
  level: 'district'
};
const district = new AMap.DistrictSearch(opts);
district.search('杭州市', (status, result) => {
  const bounds = result.districtList[0].boundaries;
  // bounds 是多边形坐标数组
});
```

#### 方式二：POI搜索获取建筑物位置（免费）
```javascript
AMap.plugin('AMap.PlaceSearch', () => {
  const placeSearch = new AMap.PlaceSearch({
    type: '建筑物',
    pageSize: 50
  });
  placeSearch.search('西湖区', (status, result) => {
    // result.poiList 包含建筑物名称和坐标
  });
});
```

#### 方式三：静态地图下载（免费，推荐）
```
https://restapi.amap.com/v3/staticmap?key=您的Key&zoom=16&size=800*600&location=120.15,30.25
```
下载后用图像处理工具手动标记建筑物/道路，再转换为网格数组。

### 3. 坐标转网格公式
```javascript
// 假设地图左上角经纬度为 (lng0, lat0)，每格代表 0.0001 度
const tileX = Math.floor((poiLng - mapLeftLng) / 0.0001);
const tileY = Math.floor((mapTopLat - poiLat) / 0.0001);
```

### 4. 数据映射规则
| 高德数据 | 转换方式 | 游戏地图类型 |
|---------|---------|-------------|
| 建筑物POI坐标 | 坐标 → 网格索引（取整除） | `obstacle` |
| 道路线数据 | 线 → 栅格化（Bresenham算法） | `floor` |
| 水域边界 | 多边形填充 | `obstacle` |
| 空白区域 | 默认 | `floor` |

### 5. 替代方案
| 平台 | 特点 | 网址 |
|------|------|------|
| **百度地图API** | POI数据丰富，建筑物轮廓较全 | [lbsyun.baidu.com](https://lbsyun.baidu.com) |
| **腾讯地图API** | 与微信生态结合好 | [lbs.qq.com](https://lbs.qq.com) |
| **天地图** | 官方权威，数据免费 | [tianditu.gov.cn](https://www.tianditu.gov.cn) |

### 6. 推荐实现流程
1. 用高德静态地图API下载目标区域俯视图
2. 用Photoshop/GIMP打开，手动标记建筑物为黑色、道路为白色
3. 编写脚本将图片像素转换为 `0`（障碍物）和 `1`（可行走）的二维数组
4. 导入 `map.js` 的 `initMap()` 中替换默认地图数据
